One-Line Installer

Install SupaVector on your machine in one command.

Pick the environment you want to run from, copy the command, and get a working supavector CLI without stepping through setup docs first.

User install System install with sudo npm global option

Requires Node.js 18+, npm, and git. Docker is only needed if you want to self-host locally after install.

Default user install
curl -fsSL https://raw.githubusercontent.com/Emmanuel-Bamidele/supavector/main/scripts/install.sh | bash

Installs the CLI under ~/.supavector and updates your user shell PATH.

Default user install
curl -fsSL https://raw.githubusercontent.com/Emmanuel-Bamidele/supavector/main/scripts/install.sh | bash

Best for local Linux development when you want a user-level install instead of a machine-wide wrapper.

Machine-wide install
curl -fsSL https://raw.githubusercontent.com/Emmanuel-Bamidele/supavector/main/scripts/install.sh | sudo bash -s -- --system

Good for EC2 or other cloud VMs when you want /usr/local/bin/supavector available for the box.

npm global install
npm install -g github:Emmanuel-Bamidele/supavector

Use this when you want npm to own the global CLI install and upgrade path.

PowerShell remote install
irm https://raw.githubusercontent.com/Emmanuel-Bamidele/supavector/main/scripts/install.ps1 | iex

Runs the Windows installer and adds the SupaVector wrapper to your user environment.

const BASE = process.env.SUPAVECTOR_BASE_URL;
const KEY  = process.env.SUPAVECTOR_API_KEY;

async function ar(path, body, extra = {}) {
  const res = await fetch(`${BASE}${path}`, {
    method:  "POST",
    headers: {
      "Authorization": `Bearer ${KEY}`,
      "Content-Type":  "application/json",
      ...extra,
    },
    body: JSON.stringify(body),
  });
  if (!res.ok) throw new Error(await res.text());
  return res.json();
}

await ar("/v1/docs",
  { docId: "welcome", collection: "default",
    text: "Supavector stores memory for agents." },
  { "Idempotency-Key": "doc-001" }
);

Self-Host And Build On Top

Up and running in minutes.

Install the CLI, onboard the stack, and write your first memory in under five minutes. Full control — your infra, your data.

01

Install the CLI

Use the one-line installer to get started fast. If you prefer working from a checkout, you can also clone the repo and run ./scripts/install.sh.

curl -fsSL https://raw.githubusercontent.com/Emmanuel-Bamidele/supavector/main/scripts/install.sh | bash

Need a machine-wide install? Click here.

02

Onboard the stack

If you used the one-line installer, onboarding brings up the vector core, gateway, and Postgres locally. If you cloned the repo instead, run the install script first, then onboard.

supavector onboard
03

Write your first memory

Use the CLI or call the API directly with your service token.

supavector write --text "Hello, memory"
04

Ask a question

Query the memory you just wrote. Supavector retrieves and answers grounded in your data.

supavector ask --question "What did I store?"
05

Add to your app env

Export the base URL and service token so your backend can reach Supavector.

export SUPAVECTOR_BASE_URL="http://localhost:3000"
export SUPAVECTOR_API_KEY="YOUR_SERVICE_TOKEN"
06

Call server-to-server

Your backend calls Supavector directly. No browser, no end-user login needed.

curl -X POST "$SUPAVECTOR_BASE_URL/v1/ask" \
  -H "X-API-Key: $SUPAVECTOR_API_KEY" \
  -d '{"question":"What did I store?","k":3}'
