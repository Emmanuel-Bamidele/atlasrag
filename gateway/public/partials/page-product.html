SUPAVECTOR_BASE_URL=https://your-host

API examples
Index a document
curl -X POST "${SUPAVECTOR_BASE_URL}/v1/docs" \
  -H "Authorization: Bearer ${SUPAVECTOR_API_KEY}" \
  -H "Idempotency-Key: doc-001" \
  -H "Content-Type: application/json" \
  -d '{
    "docId":      "welcome",
    "collection": "default",
    "text":       "Supavector stores memory for agents."
  }'
Ask a question uses credit unless you send your own provider key
curl -X POST "${SUPAVECTOR_BASE_URL}/v1/ask" \
  -H "Authorization: Bearer ${SUPAVECTOR_API_KEY}" \
  -H "Content-Type: application/json" \
  -d '{
    "question": "What does Supavector store?",
    "k":        7,
    "policy":   "amvl"
  }'
Install and create a client
python3 -m pip install supavector

from supavector import Client

client = Client.from_env(collection="default")
Index and ask ask uses credit unless you send your own provider key
client.index_text(
    "welcome",
    "SupaVector stores memory for agents.",
    params={"idempotencyKey": "doc-001"},
)

answer = client.ask("What does SupaVector store?", {"k": 7})
print(answer["data"]["answer"])
Index a document
const BASE = process.env.SUPAVECTOR_BASE_URL;
const KEY  = process.env.SUPAVECTOR_API_KEY;

async function ar(path, body, extra = {}) {
  const res = await fetch(`${BASE}${path}`, {
    method:  "POST",
    headers: {
      "Authorization": `Bearer ${KEY}`,
      "Content-Type":  "application/json",
      ...extra,
    },
    body: JSON.stringify(body),
  });
  if (!res.ok) throw new Error(await res.text());
  return res.json();
}

await ar("/v1/docs",
  { docId: "welcome", collection: "default",
    text: "Supavector stores memory for agents." },
  { "Idempotency-Key": "doc-001" }
);
Ask a question uses credit unless you send your own provider key
const { data } = await ar("/v1/ask", {
  question: "What does Supavector store?",
  k:        7,
  policy:   "amvl",
});

console.log(data.answer);

/v1/docs stays text-first by default. If you are sending source code directly, add optional fields such as sourceType: "code", title, sourceUrl, and metadata to opt that payload into code-aware chunking without changing normal text ingest.

Self-Host And Build On Top

Up and running in minutes.

Use the CLI when you want to run SupaVector yourself. Install it, onboard once, and then point your app at the local or server deployment.

01

Install the CLI

Use the one-line installer to get the supavector command on your machine.

curl -fsSL https://raw.githubusercontent.com/Emmanuel-Bamidele/supavector/main/scripts/install.sh | bash
02

Onboard the stack

This starts the local stack, creates the first admin, and saves the first service token for later CLI use.

supavector onboard
03

Store something and ask a question

Use the CLI first. That gives you a quick proof that retrieval and answer generation are working.

supavector write --text "Hello, memory"
supavector ask --question "What did I store?"
04

Add to your app env

When your app is ready, export the base URL and service token and call SupaVector server-to-server.

export SUPAVECTOR_BASE_URL="http://localhost:3000"
export SUPAVECTOR_API_KEY="YOUR_SERVICE_TOKEN"
