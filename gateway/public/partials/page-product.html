Index a document
const BASE = process.env.ATLASRAG_BASE_URL;
const KEY  = process.env.ATLASRAG_API_KEY;

async function ar(path, body, extra = {}) {
  const res = await fetch(`${BASE}${path}`, {
    method:  "POST",
    headers: {
      "Authorization": `Bearer ${KEY}`,
      "Content-Type":  "application/json",
      ...extra,
    },
    body: JSON.stringify(body),
  });
  if (!res.ok) throw new Error(await res.text());
  return res.json();
}

await ar("/v1/docs",
  { docId: "welcome", collection: "default",
    text: "AtlasRAG stores memory for agents." },
  { "Idempotency-Key": "doc-001" }
);

Self-Host And Build On Top

Up and running in minutes.

Install the CLI, start the stack, and write your first memory in under five minutes. Full control — your infra, your data.

01

Clone and install the CLI

Clone the repo, then run the installer. Works on macOS, Linux, and Windows.

git clone https://github.com/Emmanuel-Bamidele/atlasrag.git
cd atlasrag && ./scripts/install.sh
02

Start the stack

One command brings up the vector core, gateway, and Postgres locally.

atlasrag start
03

Write your first memory

Use the CLI or call the API directly with your service token.

atlasrag write --text "Hello, memory"
04

Ask a question

Query the memory you just wrote. AtlasRAG retrieves and answers grounded in your data.

atlasrag ask --question "What did I store?"
05

Add to your app env

Export the base URL and service token so your backend can reach AtlasRAG.

export ATLASRAG_BASE_URL="http://localhost:3000"
export ATLASRAG_API_KEY="YOUR_SERVICE_TOKEN"
06

Call server-to-server

Your backend calls AtlasRAG directly. No browser, no end-user login needed.

curl -X POST "$ATLASRAG_BASE_URL/v1/ask" \
  -H "X-API-Key: $ATLASRAG_API_KEY" \
  -d '{"question":"What did I store?","k":3}'

What shifts

Move memory out of app glue and into infrastructure.

Teams stop stitching together retrieval logic, ACL rules, expiry behavior, and quality loops in five different places. AtlasRAG centralizes the memory layer so product code can stay focused on the actual experience.

  • Keep your frontend, backend, and agent orchestration where they already belong.
  • Use one governed path for writes, search, ask, and memory recall.
  • Push setup, auth, API, and policy details into Documentation instead of bloating this tab.
Best fit

Customer assistants, internal copilots, research agents, and controlled automation where memory quality matters after day one.

Built in

Tenant scoping, service-token auth, retrieval policies, and lifecycle controls are already part of the platform surface.

Open Source

Built in the open. MIT licensed.

AtlasRAG is fully open source. Every line — the C++ vector core, Node.js gateway, installer scripts, and Docker Compose files — is available to inspect, fork, and self-host. There is no hosted service, no vendor lock-in, and no black-box dependencies.
