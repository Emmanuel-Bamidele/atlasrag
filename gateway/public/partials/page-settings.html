Billing

Billing

Use Dashboard for balance, payments, invoices, and add credit.

AI credit
Hosted generation uses credit unless the request sends its own provider key.
  • ask, code, and boolean_ask can use hosted credit.
  • If you send your own provider key on a supported request, generation can bill against your provider instead of hosted credit.
  • Manage add-credit and current balance in Dashboard.
Storage
Storage billing is separate from AI credit.
  • Document writes, memory writes, and retained hosted data affect storage usage.
  • Storage is billed separately from AI credit.
  • Use Dashboard for payment method, invoices, and current hosted billing state.

Use token
Store it as a secret and send it as X-API-Key from your runtime.
Rules
  • Keep service tokens server-side only.
  • Use separate tokens per environment or app.
  • Rotate or revoke tokens when an environment is retired.
Quick example
export SUPAVECTOR_BASE_URL=http://localhost:3000
export SUPAVECTOR_API_KEY=<copy-the-new-service-token>

curl -H "X-API-Key: $SUPAVECTOR_API_KEY" \
  "$SUPAVECTOR_BASE_URL/v1/collections"
