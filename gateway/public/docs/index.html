SupaVector API Documentation

This page is focused on API consumers building apps and agents. The recommended model is: bootstrap once, store a service token, then let your backend or agent call SupaVector directly.

Recommended Flow

  1. Run node scripts/bootstrap_instance.js once to create an admin and service token.
  2. Store SUPAVECTOR_BASE_URL and SUPAVECTOR_API_KEY in your app or agent env.
  3. Index content via /v1/docs or /v1/docs/url.
  4. Retrieve context with /v1/search, /v1/ask, /v1/boolean_ask, and the memory endpoints.

Usage Modes

  • Fork and self-deploy with the bundled stack for the fastest start.
  • Fork and self-deploy with your own Postgres and OpenAI key if you already have infra in place.
  • Use a shared SupaVector deployment if your app or agent just needs to call an existing instance.
  • Use a shared SupaVector deployment with your own OpenAI key when the request should use your provider key.
  • Keep your own product auth in front when SupaVector should sit behind your backend.

Important Notes

  • JWTs are best for human admin sessions. Service tokens are best for apps, agents, workers, and CI.
  • SupaVector can use your existing Postgres. Set PGHOST, PGPORT, PGDATABASE, PGUSER, and PGPASSWORD.
  • If you want SupaVector to keep its own Postgres/auth but use your OpenAI key for a request, also send X-OpenAI-API-Key.
  • Idempotency-Key is required for /v1/docs, /v1/docs/url, /v1/memory/write, /v1/memory/reflect.
  • /v1/docs/url only indexes reachable public URLs. Private/local hosts are blocked.
  • Use /v1/jobs/{id} until status becomes succeeded or failed.
  • AMV-L retrieval is tier-gated: default retrieval uses HOT + sampled WARM; COLD is excluded unless explicitly enabled.
  • Memory, ask, and boolean_ask requests can set policy to amvl, ttl, or lru. The default is amvl.
  • X-OpenAI-API-Key is request-scoped and is currently rejected by async /v1/memory/reflect and /v1/memory/compact.
  • /v1/ask supports answerLength: auto, short, medium, long.
  • /v1/boolean_ask returns only true, false, or invalid, plus supportingChunks.
Bootstrap once, then use the returned SUPAVECTOR_BASE_URL and SUPAVECTOR_API_KEY everywhere else. Add X-OpenAI-API-Key only when you want request-scoped provider override.
