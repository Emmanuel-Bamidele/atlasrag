AtlasRAG API Documentation

This page is focused on API consumers building apps and agents. It highlights only the details needed for integration and includes examples aligned with tested end-to-end flows.

Tested Integration Flow

  1. Authenticate with POST /v1/login or service token.
  2. Index content via /v1/docs or /v1/docs/url.
  3. Retrieve context with /v1/search and /v1/ask.
  4. Use memory endpoints, then poll /v1/jobs/{id} for async work.

Important Notes

  • Idempotency-Key is required for /v1/docs, /v1/docs/url, /v1/memory/write, /v1/memory/reflect.
  • /v1/docs/url only indexes reachable public URLs. Private/local hosts are blocked.
  • Use /v1/jobs/{id} until status becomes succeeded or failed.
  • AMV-L retrieval is tier-gated: default retrieval uses HOT + sampled WARM; COLD is excluded unless explicitly enabled.
  • /v1/ask supports answerLength: auto, short, medium, long.
