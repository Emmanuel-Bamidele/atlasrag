AtlasRAG provides durable memory APIs, multi-tenant governance, and retrieval workflows so teams can store, recall, and operationalize knowledge for agents in production. It combines a fast C++ vector store with a secure gateway that orchestrates embeddings, recall, and background jobs.

AtlasRAG Product Overview

AtlasRAG is a persistent memory engine for AI agents. It combines a fast vector index with structured memory APIs, multi-tenant governance, and developer-grade observability so teams can ship reliable, long-lived agents.

Memory API

Write durable memories with clear types (artifact, semantic, procedural, summary). Each memory is attributed to a tenant, collection, and principal for safe isolation.

Recall API

Query memories by vector similarity plus filters like type, time range, and collection. Recall respects visibility and ACLs for enterprise-grade access control.

Reflection & Summarization

Convert artifacts into semantic/procedural/summary memories using asynchronous jobs, with provenance links back to original sources.

Collections

Segment knowledge by product, team, or environment. Collections scope indexing, recall, and governance without creating separate deployments.

Multi-tenant Architecture

Tenant IDs are enforced at the API layer and in storage namespaces. Every vector and memory item is isolated and query-safe by default.

Governance

Fine-grained ACLs and visibility modes (tenant, private, ACL) keep sensitive knowledge constrained to the right users or service principals.

Metrics

Track ingestion throughput, vector-store performance, and gateway latency percentiles per tenant. Developers can spot hotspots before they impact production traffic.

Reliability Features

Idempotency keys, TTL cleanup, and compaction jobs reduce operational risk and keep memory stores healthy over time.

Developer Ready

OpenAPI docs, Swagger UI, and a Node SDK make it easy to integrate AtlasRAG into agents, applications, or internal platforms.

Typical Flow

  1. Index or upload artifacts into a collection.
  2. Trigger reflection jobs to produce durable memories.
  3. Recall memories with filters and ACL-aware access.
  4. Monitor metrics for latency, throughput, and quality.

Built for Teams

AtlasRAG supports multi-tenant workflows, SSO-ready authentication, and service tokens for automation. Organizations can onboard new teams without standing up separate stacks.

Quickstart

  1. Start the stack: docker compose up -d
  2. Create an admin user (first time only):
docker compose exec gateway node scripts/create_user.js \
  --username admin \
  --password changeme \
  --tenant acme \
  --role admin
  1. Login to get a JWT (required to create API keys):
curl -X POST http://localhost:3000/v1/login \
  -H "Content-Type: application/json" \
  -d '{ "username": "admin", "password": "changeme" }'
  1. Create a service token (API key) for your app:
curl -X POST http://localhost:3000/v1/admin/service-tokens \
  -H "Authorization: Bearer YOUR_ADMIN_JWT" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "app-server",
    "principalId": "svc:app",
    "roles": ["writer"],
    "expiresAt": "2026-12-31T00:00:00Z"
  }'
  1. Index your first document:
curl -X POST http://localhost:3000/v1/docs \
  -H "X-API-Key: YOUR_SERVICE_TOKEN" \
  -H "Idempotency-Key: idx-001" \
  -H "Content-Type: application/json" \
  -d '{
    "docId": "swe_notes",
    "collection": "default",
    "text": "Your document text..."
  }'
  1. Search or Ask with /v1/search and /v1/ask.

Authentication

Protected endpoints require either a JWT or a service token. Send Authorization: Bearer <jwt> or X-API-Key: <token> (also supports Authorization: ApiKey <token>).

Multi-tenant isolation is enforced by the auth token. JWTs must include a tenant identifier using one of these claims: tenant, tid, or sub. Service tokens inherit the tenant from the admin who created them.

Admins or org owners can issue service tokens via POST /v1/admin/service-tokens. Store the returned token securely; it is shown only once.

Create API key (admin)
curl -X POST http://localhost:3000/v1/admin/service-tokens \
  -H "Authorization: Bearer YOUR_ADMIN_JWT" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "ci-pipeline",
    "principalId": "svc:ci",
    "roles": ["writer"],
    "expiresAt": "2026-12-31T00:00:00Z"
  }'

Requires role admin or owner (or a token whose principal matches the tenant).

API Reference

Full OpenAPI schema is available at /openapi.json, with Swagger UI at /docs. All versioned routes live under /v1; legacy routes remain as aliases.

  • /v1/docs index + list documents
  • /v1/search retrieve top‑K chunks
  • /v1/ask RAG answers with citations
  • /v1/memory/write durable memory items
  • /v1/memory/recall filtered recall
  • /v1/memory/reflect async reflection jobs
  • /v1/jobs/:id job status

Response format (v1): all /v1 endpoints return a consistent envelope.

Success
{
  "ok": true,
  "data": { ... },
  "meta": {
    "tenantId": "acme",
    "collection": "default",
    "timestamp": "2026-02-12T12:00:00.000Z"
  }
}
Error
{
  "ok": false,
  "error": { "message": "Invalid request", "code": "INVALID_REQUEST" },
  "meta": { "tenantId": "acme", "collection": "default", "timestamp": "..." }
}

SDKs

Official SDK: sdk/node (Node.js). It supports JWT or API key auth, idempotency headers, and memory APIs. Additional SDKs can be added using the OpenAPI schema.

import os
import time
import requests

BASE = os.getenv("ATLASRAG_URL", "http://localhost:3000")
API_KEY = os.getenv("ATLASRAG_API_KEY")  # or use JWT

headers = {
  "Content-Type": "application/json",
  "X-API-Key": API_KEY
}

# 1) Index
doc = {
  "docId": "swe_notes",
  "collection": "default",
  "text": "WAL keeps vectors durable across restarts."
}
res = requests.post(f"{BASE}/v1/docs", headers={**headers, "Idempotency-Key": "idx-001"}, json=doc)
res.raise_for_status()
print(res.json())

# 2) Search
res = requests.get(f"{BASE}/v1/search", headers=headers, params={
  "q": "durability",
  "k": 5,
  "collection": "default"
})
res.raise_for_status()
print(res.json())

# 3) Ask
res = requests.post(f"{BASE}/v1/ask", headers=headers, json={
  "question": "How do we persist vectors?",
  "k": 5
})
res.raise_for_status()
print(res.json())

# 4) Memory write
res = requests.post(f"{BASE}/v1/memory/write", headers={**headers, "Idempotency-Key": "mem-001"}, json={
  "type": "semantic",
  "collection": "default",
  "text": "Vector WAL is enabled in production."
})
res.raise_for_status()

# 5) Recall
res = requests.post(f"{BASE}/v1/memory/recall", headers=headers, json={
  "query": "WAL enabled",
  "types": ["semantic"],
  "k": 5
})
res.raise_for_status()
print(res.json())

# 6) Reflect (async job)
res = requests.post(f"{BASE}/v1/memory/reflect", headers={**headers, "Idempotency-Key": "reflect-001"}, json={
  "docId": "swe_notes",
  "types": ["semantic", "summary"],
  "collection": "default"
})
res.raise_for_status()
job_id = res.json()["data"]["job"]["id"]

# 7) Poll job
while True:
  job = requests.get(f"{BASE}/v1/jobs/{job_id}", headers=headers).json()
  status = job["data"]["job"]["status"]
  if status in ("succeeded", "failed"):
    print(job)
    break
  time.sleep(2)

End-to-end flow
const BASE = process.env.ATLASRAG_URL || "http://localhost:3000";
const API_KEY = process.env.ATLASRAG_API_KEY;

const sleep = (ms) => new Promise((r) => setTimeout(r, ms));
const headers = {
  "Content-Type": "application/json",
  "X-API-Key": API_KEY
};

const post = async (path, body, extra = {}) => {
  const res = await fetch(`${BASE}${path}`, {
    method: "POST",
    headers: { ...headers, ...extra },
    body: JSON.stringify(body)
  });
  const data = await res.json();
  if (!res.ok) throw new Error(JSON.stringify(data));
  return data;
};

const get = async (path, params = {}) => {
  const url = new URL(`${BASE}${path}`);
  Object.entries(params).forEach(([k, v]) => url.searchParams.set(k, v));
  const res = await fetch(url, { headers });
  const data = await res.json();
  if (!res.ok) throw new Error(JSON.stringify(data));
  return data;
};

// 1) Index
await post("/v1/docs", {
  docId: "swe_notes",
  collection: "default",
  text: "WAL keeps vectors durable across restarts."
}, { "Idempotency-Key": "idx-001" });

// 2) Search
console.log(await get("/v1/search", { q: "durability", k: "5", collection: "default" }));

// 3) Ask
console.log(await post("/v1/ask", { question: "How do we persist vectors?", k: 5 }));

// 4) Memory write
await post("/v1/memory/write", {
  type: "semantic",
  collection: "default",
  text: "Vector WAL is enabled in production."
}, { "Idempotency-Key": "mem-001" });

// 5) Recall
console.log(await post("/v1/memory/recall", { query: "WAL enabled", types: ["semantic"], k: 5 }));

// 6) Reflect + job polling
const reflect = await post("/v1/memory/reflect", {
  docId: "swe_notes",
  types: ["semantic", "summary"],
  collection: "default"
}, { "Idempotency-Key": "reflect-001" });

const jobId = reflect.data.job.id;
while (true) {
  const job = await get(`/v1/jobs/${jobId}`);
  const status = job.data.job.status;
  if (status === "succeeded" || status === "failed") {
    console.log(job);
    break;
  }
  await sleep(2000);
}

Architecture

  • Gateway (Node) handles auth, routing, and orchestration.
  • Vector store (C++ TCP service) stores embeddings and serves similarity search.
  • Postgres stores chunks, memory items, links, jobs, and idempotency keys.
  • OpenAI (or compatible) provides embeddings and generation.
  • Background jobs handle reflection, summarization, and lifecycle tasks.

Deployment

  • Run with Docker Compose: docker compose up -d
  • Set required env: OPENAI_API_KEY, JWT_SECRET, Postgres credentials.
  • Expose gateway on :3000 and front with HTTPS in production.
  • Optionally set PUBLIC_BASE_URL / OPENAPI_BASE_URL for correct docs links.
